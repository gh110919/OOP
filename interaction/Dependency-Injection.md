### Dependency Injection (Внедрение зависимостей)

###### Dependency Injection - это способ предоставления зависимостей объектам. Он позволяет улучшить тестируемость и уменьшить связанность кода.

```
Dependency Injection (DI) — это проектировочный шаблон 
в разработке программного обеспечения,
который используется для управления зависимостями между объектами.
Основнаяцель DI заключается в том, 
чтобы уменьшить связанность компонентов системы,
упростить тестирование и улучшить масштабируемость кода.

Вместо того чтобы объект сам создавал свои зависимости,
они предоставляются ему извне. 
Это происходит через внедрение этих зависимостей,
что может быть реализовано тремя основными способами:
через конструктор, через сеттеры (методы) или через интерфейсы.
```

```
Представьте себе садовника, 
который распределяет воду из центрального источника 
к разным растениям с помощью системы шлангов. 
Вместо того, чтобы каждый раз носить воду 
к каждому растению вручную, садовник использует систему шлангов и распылителей, 
которая доставляет воду точно туда, где она нужна. 
Это позволяет садовнику легко менять и адаптировать систему полива, 
не влияя на каждое растение отдельно. Аналогично, 
внедрение зависимостей в программировании позволяет объектам 
получать необходимые ресурсы (зависимости) через внешние средства, 
улучшая тестируемость и снижая связанность кода.
```

```ts
class Engine {
  start() {
    console.log("Engine started.");
  }
}

class Car {
  private engine: Engine;

  constructor(engine: Engine) {
    this.engine = engine;
  }

  drive() {
    this.engine.start();
    console.log("Car is driving.");
  }
}

// Внедрение зависимости через конструктор
const engine = new Engine();
const car = new Car(engine);
car.drive();
```
