### ISP: Interface Segregation Principle (Принцип разделения интерфейса)

###### Многие специализированные интерфейсы лучше, чем один универсальный.

```
Представьте себе ресторан с многообразием меню. 
Вместо одного огромного меню, в котором перечислены все возможные блюда, 
есть несколько специализированных меню: 
одно для закусок, другое для основных блюд, третье для десертов и так далее. 
Это позволяет посетителям легко находить нужные блюда, 
не тратя время на изучение большого списка. 
Аналогично, принцип разделения интерфейса в программировании 
предполагает создание нескольких специализированных интерфейсов вместо одного универсального, 
чтобы каждая часть кода имела доступ только к необходимым ей методам.
```

```ts
/* Плохой пример: */
interface MultifunctionalDevice {
  print(): void;
  scan(): void;
  fax(): void;
}

class OldPrinter implements MultifunctionalDevice {
  print(): void {
    // Печать
  }

  scan(): void {
    throw new Error("Сканирование не поддерживается");
  }

  fax(): void {
    throw new Error("Факс не поддерживается");
  }
}

/* Хороший пример: */
interface Printer {
  print(): void;
}

interface Scanner {
  scan(): void;
}

interface Fax {
  fax(): void;
}

class SimplePrinter implements Printer {
  print(): void {
    // Печать
  }
}

/* Теперь SimplePrinter реализует только необходимые ему методы. */
```
